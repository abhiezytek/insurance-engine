# YPYG Output Review Guide

## Downloads in scope
- YPYG ULIP Format
- YPYG Traditional Format

## Output checks
Validate that each output supports:
- policy summary
- policy status
- premium summary
- surrender value as on date
- maturity value as on date
- maturity value at maturity assuming all future premiums are paid
- paid till date values
- future payable values
- conditional fields displayed as 0.00 or Not Applicable where required

## ULIP specific checks
- avoid mixing traditional labels into ULIP sections
- include fund-based / risk / charge-driven values only where applicable

## Traditional specific checks
- avoid ULIP specific wording like NAV / fund volatility unless explicitly required
- include surrender / maturity / survival benefit values relevant to traditional product
